你想知道除了 `+` 之外，Python 中还有哪些可以实现字符串拼接的方式，我会按「常用程度+易用性」排序，给你讲解几种主流方法，包括运算符和其他实用技巧～

### 第一步：先明确核心答案
除了 `+`，Python 中常用的字符串拼接方式还有：`*`（字符串重复+拼接）、`join()` 方法、f-string（格式化拼接）、`%` 格式化、`format()` 方法，其中 **f-string 和 join()** 是日常开发中最推荐的写法。

### 第二步：逐一讲解每种方法（附代码示例）
#### 1. `*` 运算符：字符串重复 + 拼接
`*` 本身是「字符串重复」运算符，但可以和 `+` 结合实现拼接，也可单独重复同一字符串：

```python
# 示例1：重复同一字符串（本质是特殊的拼接）
a = 'hi'
print(a * 3)  # 输出 hihihi（把 'hi' 重复3次拼接）

# 示例2：* + + 组合拼接
b = 'hello'
c = '-' * 5  # 先重复出 '-----'
print(b + c)  # 输出 hello-----
```

#### 2. `join()` 方法：批量拼接（推荐用于多字符串/列表拼接）
`join()` 是字符串的内置方法，适合把**多个字符串（或可迭代对象中的字符串）** 拼接成一个，效率比多次用 `+` 更高（尤其拼接大量字符串时）：

```python
# 语法：分隔符.join(可迭代对象)（分隔符为空则直接拼接）
# 示例1：拼接列表中的字符串（无分隔符）
str_list = ['m', 'n', '123']
result = ''.join(str_list)  # 空字符串作为分隔符
print(result)  # 输出 mn123

# 示例2：带分隔符拼接
str_list2 = ['hello', 'world', 'python']
result2 = '-'.join(str_list2)  # 用 '-' 分隔
print(result2)  # 输出 hello-world-python
```

#### 3. f-string（格式化字符串）：灵活拼接（Python3.6+ 推荐）
f-string 是最简洁、易读的拼接方式，支持直接嵌入变量、表达式，还能混合字符串和数字（自动转换）：

```python
# 示例1：拼接普通字符串变量
a = 'm'
b = 'n'
result = f'{a}{b}'  # 把 a 和 b 嵌入字符串中拼接
print(result)  # 输出 mn

# 示例2：混合字符串和数字（无需手动转类型）
name = '小明'
age = 18
result2 = f'{name}的年龄是{age}岁'
print(result2)  # 输出 小明的年龄是18岁
```

#### 4. `%` 格式化：老式拼接（兼容旧版本）
这是 Python 2 时代的主流写法，现在仍可用，但可读性不如 f-string：

```python
# 语法：'%s%s' % (变量1, 变量2)（%s 表示字符串占位符）
a = 'm'
b = 'n'
result = '%s%s' % (a, b)
print(result)  # 输出 mn

# 混合数字示例
score = 95
result2 = '分数：%s' % score  # %s 会自动把数字转字符串
print(result2)  # 输出 分数：95
```

#### 5. `format()` 方法：通用格式化拼接
比 `%` 更灵活，支持位置/关键字占位，也是 f-string 出现前的常用写法：

```python
# 示例1：位置占位
a = 'm'
b = 'n'
result = '{}{}'.format(a, b)
print(result)  # 输出 mn

# 示例2：关键字占位（更易读）
result2 = '{x}{y}'.format(x='m', y='n')
print(result2)  # 输出 mn
```

### 第三步：不同方法的使用场景对比（帮你选）
| 方法       | 优点                          | 缺点                          | 适用场景                  |
|------------|-------------------------------|-------------------------------|---------------------------|
| `+`        | 简单直观                      | 大量拼接时效率低、不能混类型  | 2-3个短字符串拼接         |
| `join()`   | 批量拼接效率高                | 需要可迭代对象（如列表）      | 拼接列表/元组中的多个字符串|
| f-string   | 简洁、易读、支持表达式/混类型 | Python3.6+ 才支持             | 绝大多数日常拼接场景（推荐）|
| `%` 格式化 | 兼容旧版本                    | 可读性差、占位符易出错        | 维护旧代码时              |
| `format()` | 灵活、兼容版本广              | 比 f-string 繁琐              | 需要兼容 Python3.5 及以下 |

### 总结
1. 日常优先用 **f-string**（简洁灵活）或 **join()**（批量拼接），替代多次使用 `+`；
2. `*` 适合「重复同一字符串」的拼接场景；
3. `%` 和 `format()` 属于老式写法，f-string 是现代 Python 的首选；
4. 所有拼接方法中，只有 `+` 和 `*` 是运算符，其余是字符串的格式化/内置方法。
