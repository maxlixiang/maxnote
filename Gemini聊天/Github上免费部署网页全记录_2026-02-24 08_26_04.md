```html
<head>

</head>
<body>
    <header class="navbar">
        <div class="logo">MyTechBlog</div>
        <nav>
            <a href="#">首页</a>
            <a href="#">关于我</a>
        </nav>
    </header>

                <a href="#" class="read-more">阅读全文 &rarr;</a>
            </article>
        </div>
    </main>

</html>
```

**代码原理解释：**

  * `class="..."` 是我们给 HTML 元素贴的“标签”，之后我们在 `style.css` 中会根据这些标签来给它们化妆（应用科技风样式）。

### 🧠 第三步：编写拉取文章的“魔法脚本”

现在我们需要告诉网页：当用户点击了某篇文章，你要去 `posts` 文件夹里把对应的 `.md` 文件拿出来，并把它翻译成 HTML。

1.  在 `my-blog` 文件夹下，新建一个文件，命名为 **`article.js`**。
2.  复制并粘贴以下代码并保存：

<!-- end list -->

```javascript
// 1. 抓取网址里的线索。比如点击第一篇文章，网址会变成 article.html?id=1，我们要把“1”抓出来
const urlParams = new URLSearchParams(window.location.search);
const articleId = parseInt(urlParams.get('id'));

// 2. 拿着抓到的 ID，去我们的数据库 (data.js) 里找找看是哪篇文章
const articleInfo = articles.find(a => a.id === articleId);

// 3. 找到网页里用来放文章的那个白板
const contentDiv = document.getElementById('markdown-content');

if (articleInfo) {
    // 动态把网页标题改成文章的标题
    document.title = articleInfo.title + " - MyTechBlog";
    
    // 4. 去 posts 文件夹把纯文本文件“拿”过来
    fetch('posts/' + articleInfo.fileName)
        .then(response => {
            if (!response.ok) {
                throw new Error('找不到文件');
            }
            return response.text(); // 把文件变成纯文本
        })
        .then(markdownText => {
            // 5. 见证奇迹：让 marked.js 把纯文本翻译成 HTML，塞进白板里！
            contentDiv.innerHTML = marked.parse(markdownText);
        })
        .catch(error => {
            contentDiv.innerHTML = '<h2>哎呀，出错了！</h2><p>找不到这篇文章的 .md 文件，请检查 posts 文件夹。</p>';
        });
} else {
    // 如果在数据库里没找到这个 ID
    contentDiv.innerHTML = '<h2>找不到该文章</h2><p>请返回首页重新选择。</p>';
}
```
