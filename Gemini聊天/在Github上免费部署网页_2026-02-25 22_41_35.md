```html
        <nav>
            <a href="#">首页</a>
            <a href="#">关于我</a>

        </div>

```

**代码原理解释：**

  * `class="..."` 是我们给 HTML 元素贴的“标签”，之后我们在 `style.css` 中会根据这些标签来给它们化妆（应用科技风样式）。

### 🧠 第三步：编写拉取文章的“魔法脚本”

现在我们需要告诉网页：当用户点击了某篇文章，你要去 `posts` 文件夹里把对应的 `.md` 文件拿出来，并把它翻译成 HTML。

```javascript
// 1. 抓取网址里的线索。比如点击第一篇文章，网址会变成 article.html?id=1，我们要把“1”抓出来
const urlParams = new URLSearchParams(window.location.search);
const articleId = parseInt(urlParams.get('id'));

// 2. 拿着抓到的 ID，去我们的数据库 (data.js) 里找找看是哪篇文章
const articleInfo = articles.find(a => a.id === articleId);

// 3. 找到网页里用来放文章的那个白板
const contentDiv = document.getElementById('markdown-content');

if (articleInfo) {
    // 动态把网页标题改成文章的标题
    document.title = articleInfo.title + " - MyTechBlog";
    
    // 4. 去 posts 文件夹把纯文本文件“拿”过来
    fetch('posts/' + articleInfo.fileName)
        .then(response => {
            if (!response.ok) {
                throw new Error('找不到文件');
            }
            return response.text(); // 把文件变成纯文本
        })
        .then(markdownText => {
            // 5. 见证奇迹：让 marked.js 把纯文本翻译成 HTML，塞进白板里！
            contentDiv.innerHTML = marked.parse(markdownText);
        })
        .catch(error => {
            contentDiv.innerHTML = '<h2>哎呀，出错了！</h2><p>找不到这篇文章的 .md 文件，请检查 posts 文件夹。</p>';
        });
} else {
    // 如果在数据库里没找到这个 ID
    contentDiv.innerHTML = '<h2>找不到该文章</h2><p>请返回首页重新选择。</p>';
}
```

### ⚙️ 第二阶段：配置 DNS 解析（这是最关键的一步）

买好域名后，你需要告诉全世界的路由器：“当有人访问 `www.我的域名.com` 时，请把他引导到 GitHub 的服务器上去。”这叫作 **DNS 解析**。

请登录你购买域名的网站，找到 **“控制台”** 或 **“域名管理”**，然后找到 **“DNS 解析”** 或 **“解析设置”** 的入口。

你需要添加 **3 条记录**（不用担心，直接照抄）：

#### 1. 添加 CNAME 记录（用于 www 开头）
这条记录的作用是：当别人输入 `www.你的域名.com` 时，自动跳转到你的 GitHub 仓库。

* **记录类型 (Type)**: `CNAME`
* **主机记录 (Name/Host)**: `www`
* **记录值 (Value/Target)**: `maxlixiang.github.io` (注意：**不要**加 `/my-blog`，只写你的用户名部分)

#### 2. 添加 A 记录（用于不带 www 的根域名）
这条记录的作用是：当别人直接输入 `你的域名.com`（不带 www）时，也能访问。GitHub 官方提供了 4 个专用 IP 地址，我们填两个就够了。

* **第一条 A 记录**：
    * 记录类型: `A`
    * 主机记录: `@` (有些地方显示为留空)

* **第二条 A 记录**：
    * 记录类型: `A`
    * 主机记录: `@`
    * 记录值: `185.199.109.153`

```bash
git fetch --all
```

```javascript
// 封装一个切换函数，方便复用
function applyTheme(theme) {
    if (theme === 'dark') {
        document.documentElement.setAttribute('data-theme', 'dark');
        themeToggle.innerHTML = '☀️';
        localStorage.setItem('theme', 'dark');
    } else {
        document.documentElement.removeAttribute('data-theme');
        themeToggle.innerHTML = '🌙';
        localStorage.setItem('theme', 'light');
    }
}

// 1. 初始化检查（保持不变）
const savedTheme = localStorage.getItem('theme');
if (savedTheme === 'dark') {
    applyTheme('dark');
}

// 2. 监听点击
themeToggle.addEventListener('click', () => {
    const isDark = document.documentElement.getAttribute('data-theme') === 'dark';
    // 如果当前是暗黑，就切到亮色；反之亦然
    applyTheme(isDark ? 'light' : 'dark');
});
```

2.  **逻辑统一**：我们将“修改属性”、“换图标”、“存本地”这三件事打包进了 `applyTheme` 函数，确保点击时这三件事同步发生，不再依赖页面刷新。

### 🛠️ 第二步：执行回退（两种常用方案）

针对你的需求，有两种专业的操作方式。作为初学者，建议先理解它们的区别：

#### 方案 A：撤销某次错误的尝试 (`git revert`) —— 最推荐
这种方式不会删除你的历史记录，而是通过**新发一次名为“撤销”的提交**来抵消之前的改动。

* **优点**：安全，不会丢失记录，适合已经传到 GitHub 的代码。
* **操作**：在终端（Terminal）输入：
    `git revert <你要撤销的那次提交的ID>`
* **结果**：你会看到暗黑模式的代码被自动删除了，但历史记录里会多一行“Revert 新增暗黑模式”。

#### 方案 B：时光倒流到特定时刻 (`git reset --hard`) —— 最果断
这种方式会直接把你的代码仓库**强制恢复**到指定的那个 Commit，之后的修改都会被“抹除”。

* **操作步骤**：
    1.  打开 Cursor 下方的终端（Terminal），如果没看到，按 `Ctrl + ~`。
    2.  输入：`git reset --hard <你想回到的那个Commit的ID>`（比如“新增关于我页面”的 ID）。
    3.  **注意**：此时你本地的代码已经变回去了，但云端 GitHub 还停留在未来。你需要执行一次**强制同步**：
        `git push origin main --force`

---

### ⚠️ 大师的温馨提示：回退的风险

* **覆盖风险**：执行 `reset --hard` 之后，你写在暗黑模式里的那些代码在本地就彻底找不到了。执行前请确认你真的不再需要它们了。
* **云端同步**：因为你之前已经成功将代码同步到了 `tiktokshopglobal.top`，普通的同步按钮可能会报错（因为它发现本地比云端“旧”）。这时必须使用上面提到的 `--force` 命令，强行让云端听从本地的指挥。
